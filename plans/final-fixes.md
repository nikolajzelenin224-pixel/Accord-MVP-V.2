## 📊 Задача #13: Упростить модуль оформления отсрочки

### Анализ:
✅ **Правильность**: АБСОЛЮТНО ВЕРНО! Логика правильная.

### Оценка:
- **Сложность**: 🟡 Средняя (20 минут)
- **Влияние**: 🔴 Критическое (Core UX)
- **Приоритет**: **P0 - Блокер**

### Текущая проблема:
- Показывается "До планового пополнения" но дата еще не выбрана
- Два варианта сбивают с толку
- Нужен только один: выбор даты

### Решение:
```jsx
// BNPLModal.jsx - упростить до одного варианта

<div>
  <h2>Оформление отсрочки</h2>
  <p>Сумма покрытия: {shortfall}</p>
  
  <label>Выберите удобную дату оплаты всех подписок</label>
  
  {/* Встроенный календарь */}
  <div className="grid grid-cols-7 gap-2">
    {days.map(day => (
      <button 
        className={day === today ? 'bg-blue-500' : 'bg-gray-50'}
        disabled={day < today}
      >
        {day}
      </button>
    ))}
  </div>
  
  {/* Детализация */}
  <div>
    <p>Сумма: {shortfall}</p>
    <p>Срок: {days} дней</p>
    <p>Комиссия: {commission}</p>
    <p>Итого: {total}</p>
  </div>
  
  <button>Оформить отсрочку</button>
</div>
```

### Улучшения:
- Убрать радиокнопки
- Показать календарь сразу
- Выделить сегодняшний день
- Заблокировать прошедшие дни
- Показать расчет динамически
