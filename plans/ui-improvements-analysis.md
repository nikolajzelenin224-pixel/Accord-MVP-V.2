## 📊 Задача #3: Блокировка скролла фона при открытых модалях

### Анализ:
✅ **Правильность**: **КРИТИЧНО!** Это базовый UX паттерн.

### Оценка:
- **Сложность**: 🟢 Низкая (15 минут)
- **Влияние**: 🔴 Критическое (UX bug)
- **Приоритет**: **P0 - Блокер**

### Решение:
```jsx
useEffect(() => {
  if (isOpen) {
    document.body.style.overflow = 'hidden';
    document.body.style.position = 'fixed';
    document.body.style.width = '100%';
  }
  return () => {
    document.body.style.overflow = '';
    document.body.style.position = '';
    document.body.style.width = '';
  };
}, [isOpen]);
```

### Улучшения:
- Создать хук `useBodyScrollLock` для переиспользования
- Сохранять позицию скролла при открытии модала
- Учитывать iOS Safari quirks

## 📊 Задача #4: Редактирование подписки при клике на тело

### Анализ:
⚠️ **Частично верно**. Нужно уточнение UX flow.

### Оценка:
- **Сложность**: 🟡 Средняя (30 минут)
- **Влияние**: 🟡 Среднее (UX)
- **Приоритет**: **P2 - Высокий**

### Проблема:
Конфликт взаимодействий:
- Клик на галочку = выбор/отмена
- Клик на тело = редактирование

### Решение:
```jsx
<div onClick={(e) => {
  e.stopPropagation();
  onEdit(subscription);
}}>
  {/* Тело подписки */}
</div>
<button onClick={(e) => {
  e.stopPropagation();
  toggleSelection();
}}>
  {/* Галочка */}
</button>
```

### Улучшения:
- Добавить иконку редактирования (карандаш) для явности
- Показывать tooltip "Нажмите для редактирования"
- Добавить swipe actions (iOS style)

### Вопросы:
❓ Что редактируется: только тариф и срок или все поля?
❓ Нужна ли возможность удаления из этого модала?
